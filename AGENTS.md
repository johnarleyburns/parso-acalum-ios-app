## What this is

Acalum: an iOS app for continuous public-domain music discovery using semantic similarity search over CLAP embeddings, MFCC, chroma, metadata, and user feedback. Users shape a single continuous stream via:

- Instrument/mood/context/era pills
- Freeform natural-language prompts
- Skips, favorites, and listening duration

No accounts, no server dependency (local vector DB), offline-capable with downloads.

## Stack

- Swift + SwiftUI, iOS 17+
- **AVFoundation** (AVPlayer + AVAudioSession) for audio playback
- **MediaPlayer** (MPRemoteCommandCenter + MPNowPlayingInfoCenter) for lock screen / background audio
- **Core ML** for CLAP text embedding (local on-device)
- **SQLite** via `parso_indexer.db` (copied from sibling project at build time)
- **Accelerate** (vDSP) for vector math
- **XcodeGen** (`project.yml`) — never edit `.xcodeproj` directly
- **Combine** for reactive state (avoid introducing RxSwift/async algorithms)

| `Resources/` | `Assets.xcassets`, `parso_indexer.db` |

- **Mood matching pipeline:** CLAP cosine similarity → pill/tag text matching → weighted fusion → sigmoid calibration → 0-100 mood index.
- **Calibration:** `MoodIndexCalibrator(sLo: 0.18, pLo: 0.12, sHi: 0.80, pHi: 0.92)` — raw score 0.18 maps to ~12, 0.80 maps to ~92.
- **Background audio:** Requires `UIBackgroundModes: [audio]` in `Info.plist` (managed via `project.yml` → `info` block). `AVAudioSession` category `.playback`, remote commands must have `isEnabled = true`.
- **Info.plist** is generated by XcodeGen via the `info:` block in `project.yml`. Do not hand-edit the plist; edit `project.yml` and re-run `xcodegen`.
- Database (`parso_indexer.db`) is pre-built by the `parso-ia-music-indexer` sibling project and copied at build time via a pre-build script.

## Commands

- Gen project: `xcodegen`
- Build: `xcodebuild -project Acalum.xcodeproj -scheme Acalum -destination 'platform=iOS Simulator,id=FC7B2F90-A27B-4BD5-9313-7B267636E165' build`
- Test: same with `test` instead of `build`; add `-only-testing:AcalumTests/<TestClass>` to run a subset
- Real-device builds required to test background audio, interruptions, and lock screen behavior (simulator does not handle audio sessions realistically).

## Post-task checklist (ALWAYS run after completing a task)

1. **Audit plan vs implementation** — re-read the plan (if one exists) and check every acceptance criterion, data-model change, and UI requirement against the shipped code. Fix any discrepancies before committing.
2. **Update `current_state.md`** with what shipped, new test counts, and any intentional deviations from the plan.
3. **Update README** if the feature changes user-visible behavior, setup steps, or the high-level description of the app.
4. **Stage + commit** all changes with a concise, descriptive commit message. Include changed files; do not commit untracked design artifacts.
5. **Merge to `main`** (already on `main` for direct work; for branch work: `git checkout main && git merge --ff-only <branch>`).
6. **Push** (`git push origin main`).
